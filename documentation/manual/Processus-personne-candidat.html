Définition

Personne : entité correspondant à une personne réutilisable dans le système (p.ex. pour la composition d'une commission). Il est possible de gérer des contrats et suivre les évolutions de sa fiche administrative (contrats, formations, etc.).

Candidat: entité utilisée en principe qu'une fois dans une commission en tant que candidat.

Problème

Dans le système, les membres d'une commission sont de type 'personne', les candidats sont de type 'candidat'. Elles ne sont pas liées de manière forte, c'est-à-dire qu'il n'est à priori pas possible de savoir quelle personne a été candidate dans quelle commission.

Raisons de ce choix

90% des candidats ne sont pas retenus. La séparation en deux entités différentes permet des meilleures performances d'accès à la base de données. Dans le cas où toutes les informations sont réunions, pour chaque recherche les résultats non pertinentes sont écartés par le système. Plus il y a d'éléments à trier, plus l'accès est lent.

Solution

  • Dans le cas où une personne existante dans la gestion des personnes devient candidat dans une commissions, il est possible d'envisager la possibilité de créer un nouveau candidat en copiant les informations existantes. Un lien existerait donc à ce moment-là.
  • Il est possible qu'un candidat qui accepte une proposition d'engagement devienne une personne. Les informations seront donc recopiées dans ce cas-là.
  • Il est possible de rechercher une personne parmi une liste de tous les candidats de toutes les commissions pour trouver une ou plusieurs occurrences.

Dans tous les cas, le lien se fait manuellement et volontairement par l'utilisateur. Le système ne garantit pas la justesse de l'information à 100%.
