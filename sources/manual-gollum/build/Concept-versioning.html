Concept versioning

Aspect technique

A chaque modification, une version est créée.

Métadonnées sur les versions

Les métadonnées permettent faire le lien entre une version et toutes les entités (modèles) impactés via une ou plusieurs jointures.

Par exemple, si je cherche toutes le modifications sur une personne, je veux également pouvoir retrouver toutes les modifications sur les entités liées à cette personnes: personne_adresse, adresse, personne_email, personne_formation, personne_activite, ...

Aspect ergonomique

Versions mineures/majeures

La plupart des versions ne sont pas significatives. Seules certaines versions le sont. Exemples?

Solutions

  • Commentaire pour version: si un commentaire de version est entré à l'enregistrement, alors la version est considérée comme majeure
  • Système de tags (ala SVN, GIT): l'utilisateur peut créer un jalon (tag) sur une entité à n'importe quel moment. Cela crée une version (virtuelle car vide: ne contenant pas de jeu de modifications), qui permet d'afficher l'état de l'entité au temps t de la création du jalon (tag).
